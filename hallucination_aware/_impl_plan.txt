Overview
This plan covers all remaining work to bring the system from a fully-coded-but-uninitialized state to a running, tested, and demo-ready product. All 6 core modules are implemented — what's missing is data artifacts, benchmark integration, tests, and CI/CD.

Work is split into 3 strictly sequential phases. Each person completes their phase, pushes to GitHub, and the next person clones/pulls the updated repo before starting.

How to Train the Models
The training logic is inside hallucination_predictor.py. Run these commands from a Python REPL or save as a one-off script:

from modules import load_config
from modules.hallucination_predictor import HallucinationPredictor

config = load_config()
predictor = HallucinationPredictor(config)

# This loads data/labeled_dataset.jsonl, trains, evaluates, and saves both .pkl files
metrics = predictor.train('data/labeled_dataset.jsonl')
print(metrics)   # Should show AUROC and F1 scores

ℹ If AUROC is below 0.65, re-run label_dataset.py with a higher --n (try 500) and retrain. The heuristic fallback is fine for smoke testing but benchmarks in Phase 2 need real models.

Tasks

Converter Script Specs
scripts/convert_truthfulqa.py
Input: data/truthfulqa_raw.jsonl (HuggingFace format)
Output format — each line of data/truthfulqa.jsonl:
{ "query": "<question>", "correct_answer": "<best_answer>", "incorrect_answers": ["...", ...], "label": 0 or 1 }
label = 1 if the question is likely to cause hallucination (use question category or incorrect_answers count as proxy).

scripts/convert_fever.py
Input: data/fever_raw.jsonl (HuggingFace format)
Output format — each line of data/fever.jsonl:
{ "query": "<claim>", "verdict": "SUPPORTS" | "REFUTES" | "NOT ENOUGH INFO", "evidence": "<evidence text>", "label": 0 or 1 }
label = 1 if verdict is REFUTES (high hallucination risk).

evaluate_benchmarks.py Spec
This script should:
Load data/truthfulqa.jsonl and data/fever.jsonl
Run each query through the full pipeline (Pipeline.run(query))
Compare pipeline verdict against ground-truth label
Compute: accuracy, hallucination detection rate, false positive rate, strategy distribution
Save results to data/benchmark_results.json in this format:
{ "truthfulqa": { "accuracy": 0.0, "hallucination_rate": 0.0, "strategy_dist": {} },
  "fever": { "accuracy": 0.0, "hallucination_rate": 0.0, "strategy_dist": {} } }

⚠ Person 3 will write tests against this exact JSON structure. Do not change the top-level keys.

Decision: FAISS Extension Strategy
Use the --input flag approach on build_index.py (not a separate indexer). This keeps a single code path for index building and avoids Person 3 having to test two different scripts. Concrete change:
python scripts/build_index.py --input data/truthfulqa.jsonl data/fever.jsonl

test_pipeline.py Spec

Tests to include:

ci.yml Spec
File: .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with: { python-version: '3.10' }
      - run: pip install -r requirements.txt
      - run: python -m pytest tests/ -v
      - run: pip install ruff && ruff check src/ modules/ scripts/ --ignore E501

ℹ Use ruff (not flake8) — it's faster and requires no extra config file. The --ignore E501 flag skips line-length warnings so you don't spend time reformatting existing code.
